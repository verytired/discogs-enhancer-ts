# 開発フロー

このプロジェクトでは、以下の開発フローを採用しています。

## ブランチ戦略

1. **Master Branch (`master` / `main`)**
   - 常にリリース可能な安定版の状態を保ちます。
   - 直接のコミットは避け、リリース時にのみマージされます。

2. **Release Branch (`release/vX.X.X`)**
   - 次期リリースのために作成される中間ブランチです。
   - 開発中の機能追加やバグ修正は、このブランチに対してマージします。
   - リリース準備が整うまで、このブランチで統合・テストを行います。

3. **Feature Branch (`feat/xxx`, `fix/xxx` 等)**
   - 個別のIssue対応や機能実装のために作成します。
   - 作業完了後、対応する **Release Branch** にマージします。

## リリース手順

1. `release/vX.X.X` ブランチで最終確認を行います。
2. `release/vX.X.X` を `master` にマージします。
3. バージョンタグを作成し、GitHubへプッシュします。
4. GitHubのReleasesページで、作成したタグに対してリリースノートを作成・公開します。
